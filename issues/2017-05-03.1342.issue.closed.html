#1342 Issue closed: Error: "No files to copy (RESULT_FILES is empty)" when using TSM + PXE¶

Labels: bug, fixed / solved / done

schabrolles opened issue at 2017-05-03 16:10:¶

During mkrescue phase I got the error :
Error: "No files to copy (RESULT_FILES is empty)"

output/TSM/default/950_dsmc_save_result_files.sh try to backup RESULT_FILES, bit those one seems to be removed by previous script : output/TSM/default/800_copy_tftp.sh

=> $RESULT_FILES is empty, then we got the error message.

950_dsmc_save_result_files.sh should not run if $PXE_TFTP_URL is set
