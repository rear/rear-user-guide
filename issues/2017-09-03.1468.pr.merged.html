#1450 introduced a function (apply-mappings) in usr/share/rear/lib/layout-functions.sh ( f408a6936388ecef39ba0780ec514394b5db5cbe and 7a76d5edc06ab7ce1750347222b7d192507d690a)

The aim of this function is to avoid call to /layout/prepare/default/320_apply_mappings.sh, but
apply-mappings() function with parameter instead.

tmp_layout="$LAYOUT_FILE"
LAYOUT_FILE="$OLD_ID_FILE"
source $SHARE_DIR/layout/prepare/default/320_apply_mappings.sh
LAYOUT_FILE="$tmp_layout"

replace by:

apply_mappings $OLD_ID_FILE

This PR finish the job by redirecting all calls to 320_apply_mappings.sh to the new apply-mappings() function.
