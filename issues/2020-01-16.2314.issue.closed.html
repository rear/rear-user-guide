  • Description of the issue (ideally so that others can reproduce it):

After creating an ISO from the server (Dell PowerEdge R830), I burned the ISO to a dvd and since I couldn't test to see if the image would boot and bring up the ReaR Menu, I attempted to boot the ISO on an older Dell OptiPlex 990. The R830 has 32G ram and the Optiplex only has 16G ram.

When booting the ISO image, the boot would fail during "Loading initrd.cgz" with the error:
"Not enough memory to load specified image". I tried another DVD ISO from another R830 in which I did successfully perform a BMR but I received the same message.

  • Workaround, if any:

Before actually booting the image, edit the boot line and add (without the quotes) "mem=16000m" and the image was able to load. I used 16000m since my Optiplex has 16G of ram.
