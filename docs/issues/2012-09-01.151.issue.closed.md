On a freshly installed Debian sid / wheezy recovery fails with the error
message  
No code has been generated to restore device fs:/

To reproduce the error following steps are necessary:

-   Install Debian sid / wheezy in a virtual environment with two disks.
    The first one is used for the OS, the second one for backup
-   Install rear
-   Produce iso and backup (BACKUP=NETFS) with "rear mkbackup"
-   Boot with ISO into second virtual machine with same disk structure
-   rear recover stops with above mentioned error message and a prompt
    saying  
    Please add code to /var/lib/rear/layout/diskrestore.sh

The reason seems to be the device naming of devices mounted by UUID. For
a device entry  
UUID=28f657a5-5242-46df-9fd4-87060cd43aa2 / ext4 errors=remount-ro 0 1  
in /etc/fstab mount produces since Debian wheezy an output of  
/dev/disk/by-uuid/28f657a5-5242-46df-9fd4-87060cd43aa2 on / type ext4
(rw,relatime,errors=remount-ro,user\_xattr,barrier=1,data=ordered)  
This breaks the dependency chain between FS and partition in
disklayout.conf because devices named /dev/disk/by-uuid/\* are not
handled appropriately. Change
[https://github.com/rear/rear/pull/152](https://github.com/rear/rear/pull/152)
should fix it.
